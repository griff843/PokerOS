# Roadmaps Docs

Canonical planning area for Poker OS roadmaps, requirements, and gap tracking.

## Files

| File | Role |
|------|------|
| `GITHUB_EXECUTION_MODEL.md` | **Active execution control plane.** Governs issues, PRs, labels, milestones. Read this first. |
| `INITIAL_GITHUB_BACKLOG.md` | First 11 execution-ready issue drafts. Source for creating GitHub issues. |
| `MASTER_ROADMAP.md` | Phase definitions (1–5). **Execution reference** for milestone assignments. |
| `COACH_EQUIVALENCE_ROADMAP.md` | Coach-level product direction (Phases 7–12). Product direction, not execution scheduling. |
| `COACH_EQUIVALENCE_REQUIREMENTS.md` | Capability requirements for coach-equivalent quality. |
| `COACH_EQUIVALENCE_GAP_TRACKER.md` | Current capability status. Updated as issues close. See stale-row notices inside. |
| `LIVE_CASH_READINESS_SPRINTS.md` | Sprint structure for live-cash product readiness. |

## Which doc governs what

- **Execution order and issue creation:** `GITHUB_EXECUTION_MODEL.md` + `INITIAL_GITHUB_BACKLOG.md`
- **Milestone assignments:** `MASTER_ROADMAP.md` Phases 3–5
- **Product direction and coaching depth targets:** `COACH_EQUIVALENCE_ROADMAP.md`
- **Current capability status:** `COACH_EQUIVALENCE_GAP_TRACKER.md` (verify against code before trusting)

## Notes

- This directory is the canonical home for roadmap documents.
- Avoid creating new planning docs under `docs/roadmap`.
